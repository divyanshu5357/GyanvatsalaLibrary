# 🚀 RENDER DEPLOYMENT GUIDE - Gyanvatsala Library

## Prerequisites

- GitHub account with your repository pushed: https://github.com/divyanshu5357/GyanvatsalaLibrary.git
- Render account: https://render.com (free tier available)
- Supabase project with API keys
- Cloudinary account (optional, for image uploads)
- Generate VAPID keys for push notifications (or skip if not needed)

---

## Step 1: Generate VAPID Keys (For Push Notifications)

If you want push notifications to work, generate VAPID keys:

### Using online tool:
1. Visit: https://web-push-codelab.glitch.me/
2. Click "Generate Keys"
3. Copy both public and private keys

### Or using Node.js locally:
```bash
npm install -g web-push
web-push generate-vapid-keys
```

Save the output - you'll need these in Render.

---

## Step 2: Connect Your GitHub Repository to Render

1. Go to: https://dashboard.render.com
2. Click **"New +"** button
3. Select **"Multi-Service Blueprint"** (or "Web Service" if not available)
4. Click **"Connect Repository"**
5. Search for: `GyanvatsalaLibrary`
6. Click **"Connect"**
7. Render will detect your `render.yaml` automatically

---

## Step 3: Configure Environment Variables in Render

Render will create 2 services automatically based on `render.yaml`:
1. **gyanvatsala-library-api** (Node.js backend)
2. **gyanvatsala-library-web** (React frontend)

### For Backend Service (gyanvatsala-library-api):

1. Go to **gyanvatsala-library-api** service
2. Click **"Settings"** → **"Environment"**
3. Add these environment variables:

```
VITE_SUPABASE_URL = https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY = your-anon-key-from-supabase
SUPABASE_SERVICE_ROLE_KEY = your-service-role-key
VAPID_PUBLIC_KEY = your-vapid-public-key (from Step 1)
VAPID_PRIVATE_KEY = your-vapid-private-key (from Step 1)
CLOUDINARY_CLOUD_NAME = your-cloudinary-name (optional)
CLOUDINARY_UPLOAD_PRESET = your-cloudinary-preset (optional)
```

**Where to get these values:**

| Variable | Source |
|----------|--------|
| VITE_SUPABASE_URL | Supabase Dashboard → Settings → API → Project URL |
| VITE_SUPABASE_ANON_KEY | Supabase Dashboard → Settings → API → anon public key |
| SUPABASE_SERVICE_ROLE_KEY | Supabase Dashboard → Settings → API → service_role key |
| VAPID_PUBLIC_KEY | Generated in Step 1 |
| VAPID_PRIVATE_KEY | Generated in Step 1 |
| CLOUDINARY_CLOUD_NAME | Cloudinary Dashboard → Settings → Account |
| CLOUDINARY_UPLOAD_PRESET | Cloudinary Dashboard → Settings → Upload |

### For Frontend Service (gyanvatsala-library-web):

1. Go to **gyanvatsala-library-web** service
2. Click **"Settings"** → **"Environment"**
3. Add these environment variables:

```
VITE_SUPABASE_URL = https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY = your-anon-key-from-supabase
VITE_CLOUDINARY_CLOUD_NAME = your-cloudinary-name (optional)
VITE_CLOUDINARY_UPLOAD_PRESET = your-cloudinary-preset (optional)
```

## Troubleshooting

### Backend fails to build
**Error**: `npm ERR! gyp ERR!`
- **Solution**: Check if all dependencies in `server/package.json` are installed
- Run locally: `cd server && npm install`
- Push changes: `git add . && git commit -m "fix deps" && git push`

### Frontend shows "Cannot connect to API"
**Error**: `Failed to fetch from /api/...`
- **Solution**: 
  - Check backend URL in frontend environment variables
  - Verify backend is running (check backend service logs)
  - Check CORS configuration on backend

### Environment variables not working
**Error**: `undefined` or config missing
- **Solution**:
  - Verify variables are set in Render dashboard (not in `.env` files)
  - Redeploy services after adding variables:
    - Go to service → **"Settings"** → **"Deploy"** → Click **"Manual Deploy"**

### 502 Bad Gateway
**Error**: Gateway Error when accessing frontend
- **Solution**:
  - Wait 30-60 seconds (initial boot time)
  - Check if backend is deployed and running
  - Check backend service logs for errors

## Important Notes

### About Port Numbers
- **Local development**: Backend on port 3001
- **Render**: Backend on port 10000 (configured in render.yaml)
- **Frontend**: No specific port (static site served on Render's infrastructure)

### About Environment Files
- `.env` files are NOT committed to GitHub (in `.gitignore`)
- Environment variables are ONLY set in Render dashboard
- No `.env` file needed in Render - values set in UI

### About Build Times
- First build: 2-5 minutes
- Subsequent builds: 1-2 minutes
- Free tier might be slower

### About Availability
- Free tier services go to sleep after 15 minutes of inactivity
- You'll see ~30s delay on first request (wake-up time)
- Paid plans are always active
